Week 3: Applying BERT to Historical Documents — Clustering and the Semantics of ‘Credit’

Overview

Over the past two weeks, we built up a toolkit: we learned how BERT produces contextual embeddings (Week 1), and we fine-tuned BERT on the Geneva Bible to teach it Early Modern English (Week 2). Today, we put that toolkit to use on a real historical archive.

We have 626 mercantile documents — letters, contracts, petitions, and commercial records from the early modern period. We are going to:

  1. Cluster the documents to discover what kinds of texts the archive contains
  2. Investigate the word “credit” across the corpus, asking: in which documents does “credit” mean social reputation and trustworthiness, and in which does it mean a financial instrument or debt?

This second question is not just a technical exercise. The entanglement of social and financial meanings of “credit” is one of the central stories of early modern economic culture. Reputation was capital; a merchant’s word was his credit line. The gradual separation of these meanings tracks a fundamental transformation in how economic life was organized. BERT can help us see where that separation is already happening — and where it is not.

By the end of this tutorial, you will be able to:

  1. Embed documents using a fine-tuned BERT model
  2. Cluster embeddings and interpret the results
  3. Extract word-level contextual embeddings from specific passages
  4. Analyze how the same word carries different meanings across a corpus

Part II — Document Embedding

To cluster the documents, we need to represent each one as a single vector. Since most documents are longer than BERT’s 512-token limit, we use a chunking and averaging strategy:

  1. Split each document into chunks of ~512 tokens
  2. Embed each chunk separately
  3. Average the chunk embeddings to get one vector per document
```{python}
def embed_document(text, tokenizer, model, max_length=512, stride=256):
    """
    Embed a long document by chunking with overlap and averaging.
    
    Args:
        text: the document text
        tokenizer: BERT tokenizer
        model: BERT model
        max_length: maximum tokens per chunk (BERT limit = 512)
        stride: overlap between chunks (helps with boundary effects)
    
    Returns:
        A single 768-dimensional vector representing the document.
    """
    # Tokenize the full document
    tokens = tokenizer.encode(text, add_special_tokens=False)
    
    if len(tokens) == 0:
        return None
    
    # Create overlapping chunks
    chunks = []
    for start in range(0, len(tokens), stride):
        chunk = tokens[start:start + max_length - 2]  # Leave room for [CLS] and [SEP]
        if len(chunk) < 10:  # Skip very short final chunks
            continue
        # Add special tokens
        chunk = [tokenizer.cls_token_id] + chunk + [tokenizer.sep_token_id]
        chunks.append(chunk)
    
    if not chunks:
        return None
    
    # Embed each chunk and average
    chunk_embeddings = []
    for chunk in chunks:
        input_ids = torch.tensor([chunk]).to(device)
        attention_mask = torch.ones_like(input_ids).to(device)
        
        with torch.no_grad():
            outputs = model(input_ids=input_ids, attention_mask=attention_mask)
        
        # Mean pooling over all tokens (excluding [CLS] and [SEP])
        hidden = outputs.last_hidden_state[0, 1:-1, :]  # Remove special tokens
        chunk_emb = hidden.mean(dim=0).cpu().numpy()
        chunk_embeddings.append(chunk_emb)
    
    # Average across chunks
    return np.mean(chunk_embeddings, axis=0)
```

# Find all occurrences of "credit" (including spelling variants)
credit_pattern = re.compile(r'\b(credit|creditt|credite|creditts?|credits?)\b', re.IGNORECASE)

credit_occurrences = []

for i, doc in enumerate(documents):
    # Find all sentences containing "credit"

Part V — Extracting Contextual Embeddings for “Credit”

This is the heart of the analysis. For each occurrence of “credit,” we extract BERT’s contextual embedding of that specific token in its specific sentence context. The same word will have different embeddings depending on whether the surrounding text is about financial transactions or about a person’s reputation.

```{python}
def get_word_embedding(sentence, target_word, tokenizer, model):
    """
    Extract the contextual embedding for a specific word in a sentence.
    
    If the word is split into multiple subword tokens by BERT's tokenizer,
    we average their embeddings to get a single word-level representation.
    """
    inputs = tokenizer(sentence, return_tensors="pt", truncation=True, max_length=512)
    inputs = {k: v.to(device) for k, v in inputs.items()}
    tokens = tokenizer.convert_ids_to_tokens(inputs["input_ids"][0].cpu())
    
    # Reconstruct words from subword tokens to find our target
    target_lower = target_word.lower()
    target_indices = []
    current_word = ""
    current_indices = []
    
    for idx, token in enumerate(tokens):
        if token in ["[CLS]", "[SEP]", "[PAD]"]:
            if current_word == target_lower and current_indices:
                target_indices = current_indices
                break
            current_word = ""
            current_indices = []
            continue
        
        if token.startswith("##"):
            current_word += token[2:]
            current_indices.append(idx)
        else:
            if current_word == target_lower and current_indices:
                target_indices = current_indices
                break
            current_word = token
            current_indices = [idx]
    
    if not target_indices and current_word == target_lower and current_indices:
        target_indices = current_indices
    
    if not target_indices:
        return None
    
    with torch.no_grad():
        outputs = model(**inputs)
    
    hidden_states = outputs.last_hidden_state[0].cpu()
    word_embedding = hidden_states[target_indices].mean(dim=0).numpy()
    
    return word_embedding
```

Part VIII — The Borderline Cases

Let’s specifically look for the sentences where BERT is most uncertain — the ones closest to the boundary between clusters.

```{python}
# Compute each point's distance to both cluster centroids
centroids = km_credit.cluster_centers_
distances_to_centroids = np.array([
    [np.linalg.norm(emb - centroid) for centroid in centroids]
    for emb in credit_embeddings
])

# "Ambiguity score": how close the two distances are (small = ambiguous)
ambiguity = np.abs(distances_to_centroids[:, 0] - distances_to_centroids[:, 1])

# Find the most ambiguous cases
most_ambiguous = np.argsort(ambiguity)[:10]

print("=== Most Ambiguous Uses of 'Credit' ===")
print("(Closest to the boundary between clusters)\n")

for rank, idx in enumerate(most_ambiguous, 1):
    meta = credit_metadata[idx]
    cluster = credit_cluster_labels[idx]
    d0, d1 = distances_to_centroids[idx]
    
    print(f"{rank}. [Cluster {cluster}] Ambiguity: {ambiguity[idx]:.4f}")
    print(f"   Distance to Cluster 0: {d0:.4f} | Distance to Cluster 1: {d1:.4f}")
    print(f"   [{meta['filename']}]")
    print(f"   {meta['sentence'][:250]}")
    print()
```

Why the Borderline Cases Matter

In early modern England, “credit” was inherently dual. A merchant’s financial creditworthiness was inseparable from his social reputation. When someone said a merchant “had credit,” they could mean he was trusted in the community, or that he could obtain goods on account, or both simultaneously — because those were the same thing.

The sentences BERT places at the boundary between clusters may be the ones where this historical duality is most visible. These are not cases where BERT “got it wrong.” They are cases where the word genuinely participates in both meanings at once. BERT’s uncertainty is, in this case, historically accurate.

  • If the social and economic senses are evenly distributed across all document types, that suggests the duality is pervasive across the archive.

What the “Credit” Analysis Reveals

The clustering of “credit” embeddings is not just a technical exercise. It tells us something about the history of the concept:

  • If the clusters are clean and well-separated, it suggests that even in this period, the social and economic senses of “credit” were already distinguishable in how writers used the word.
  • If the clusters are messy with many borderline cases, it suggests the two meanings were still deeply entangled — which is what historians like Craig Muldrew have argued was characteristic of early modern commercial culture.
  • The distribution across document types tells us where the separation was happening first. Commercial contracts might already be using “credit” in a narrowly financial sense, while personal letters still used it to mean reputation.

BERT does not answer these questions for you. It surfaces the patterns and points you to the right documents. The interpretation is yours.

Limitations

Be honest about what this analysis can and cannot do:

  • BERT’s clusters are suggestive, not definitive. Always read the actual passages.
  • Fine-tuning on the Geneva Bible helps with period language generally, but the Bible is not commercial prose. A model fine-tuned on mercantile texts directly would likely perform better.
  • Our sentence extraction is crude (splitting on punctuation). Better sentence segmentation would improve results.
  • Spelling variation is still a challenge. We searched for “credit/creditt/credite” but may have missed other forms.
  • The two-cluster (social vs. economic) framing is a simplification. There may be more than two senses.
