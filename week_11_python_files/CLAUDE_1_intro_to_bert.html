Week 1: How BERT Reads — From Tokens to Embeddings

Overview and Goals

In previous weeks, we represented text using increasingly sophisticated methods:

  • Character-level features (edit distance)
  • Token-level features (DFM + TF-IDF)
  • Word embeddings (Word2Vec: one vector per word)

Today we take the next step: contextual embeddings using BERT (Bidirectional Encoder Representations from Transformers).

By the end of this tutorial, you will be able to:

  1. Explain the difference between static and contextual embeddings
  2. Understand how BERT tokenizes text (and why it matters)
  3. Extract sentence-level embeddings using BERT
  4. Compare sentence similarity using both TF-IDF and BERT
  5. Visualize embedding spaces with PCA
  6. See what BERT “knows” through masked language modeling

We will work with a small, hand-built toy corpus so that every result is interpretable. Next week, we scale up.

Part I — From Word2Vec to BERT: The Conceptual Shift

Static Embeddings Recap

Word2Vec assigns one vector per word type. The word “company” gets a single representation regardless of whether it appears in “the East India Company” or “I enjoy her company.” This is a real limitation when words have multiple senses.

Contextual Embeddings

BERT assigns one vector per token occurrence. The embedding for “company” depends on the surrounding words in that specific sentence. “The East India Company traded spices” and “She was pleasant company at dinner” will produce different vectors for the same word.

This is the key conceptual shift: meaning is not a property of a word in isolation, but of a word in context.

How BERT Learns

BERT was pre-trained on a massive corpus (Wikipedia + BookCorpus) using two tasks:

  1. Masked Language Modeling (MLM): randomly mask 15% of tokens in a sentence, then predict what the masked tokens were. This forces BERT to build representations that encode the full context of each token.

  2. Next Sentence Prediction: given two sentences, predict whether the second follows the first in the original text.

The result of this training is a model that encodes deep knowledge about how English works — word meanings, syntax, common associations. We can then extract and use those representations for our own purposes.

BERT’s Architecture (Conceptual)

At a high level, BERT processes text in three stages:

  1. Tokenization → the input sentence is broken into subword tokens
  2. Self-attention layers → each token’s representation is updated based on every other token in the sentence (12 layers in bert-base)
  3. Output → a 768-dimensional vector for each token, encoding its meaning in context

These output vectors are the contextual embeddings we will work with.

We will build a small corpus by hand. The sentences are designed to test specific things: paraphrase detection, spelling variation, and semantic similarity across different phrasings.

```{python}
sentences = [
    # Group 1: Virginia Company — same content, different phrasing/spelling
    "The Virginia Companie did advance traffique beyond the seas.",
    "The Virginia Company did advance traffic beyond the seas.",
    "The Virginia Company promoted overseas trade.",
    
    # Group 2: Merchant activity

    # Group 3: Unrelated (control)
    "The harvest was plentiful this year.",

# Labels for reference
labels = [
    "VA Co. (archaic spelling)",
    "VA Co. (modern spelling)",
    "VA Co. (paraphrase)",
    "Merchants (version A)",
    "Merchants (version B)",
    "Harvest (unrelated)",
    "Parliament (unrelated)",
]
```

Take a moment to think about what you expect:

  • Sentences 0, 1, and 2 say the same thing. Should their similarity be high?
  • Sentences 3 and 4 are paraphrases of each other.
  • Sentences 5 and 6 are about completely different topics.

Part IV — Baseline: TF-IDF Similarity

Before we use BERT, let’s establish a baseline with TF-IDF. This represents each sentence as a sparse vector of word frequencies, weighted by how distinctive each word is.

```{python}
vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(sentences)

tfidf_sim = cosine_similarity(tfidf_matrix)

# Display as a formatted table
print("TF-IDF Cosine Similarity Matrix:\n")
print(f"{'':>30s}", end="")
for i in range(len(sentences)):
    print(f"  [{i}]", end="")
print()

for i in range(len(sentences)):
    print(f"{labels[i]:>30s}", end="")
    for j in range(len(sentences)):
        print(f" {tfidf_sim[i][j]:5.2f}", end="")
    print()
```

What to notice

Look at the TF-IDF similarity between sentences 0 and 2 (“The Virginia Companie did advance traffique beyond the seas” vs. “The Virginia Company promoted overseas trade”). These sentences mean the same thing, but because they use mostly different words, TF-IDF gives them a relatively low similarity score.

Also look at sentences 0 and 1: the only difference is spelling (“Companie” → “Company”, “traffique” → “traffic”), but TF-IDF treats these as completely different tokens. This is a fundamental limitation of bag-of-words methods for historical text.

Part V — BERT Sentence Embeddings

  • Create file: step2_bert_similarity.py — BERT sentence embeddings + comparison (Parts V–VI)

Now let’s see how BERT handles the same sentences. We will use sentence-transformers, which wraps a BERT-based model optimized for computing sentence similarity.

```{python}
# Load a sentence-transformer model
# all-MiniLM-L6-v2 is a lightweight, fast model good for similarity tasks
model = SentenceTransformer("all-MiniLM-L6-v2")

# Encode all sentences into 384-dimensional vectors
embeddings = model.encode(sentences)

print(f"Embedding shape: {embeddings.shape}")
print(f"Each sentence is now a vector of {embeddings.shape[1]} numbers.\n")

# Compute BERT-based similarity
bert_sim = cosine_similarity(embeddings)

print("BERT Cosine Similarity Matrix:\n")
print(f"{'':>30s}", end="")
for i in range(len(sentences)):
    print(f"  [{i}]", end="")
print()

for i in range(len(sentences)):
    print(f"{labels[i]:>30s}", end="")
    for j in range(len(sentences)):
        print(f" {bert_sim[i][j]:5.2f}", end="")
    print()
```

What to notice

Compare these numbers to the TF-IDF matrix:

  • Paraphrase detection: sentences 0, 1, and 2 should now show much higher similarity, because BERT recognizes that “advance traffique beyond the seas” and “promoted overseas trade” mean the same thing.
  • Spelling robustness: “Companie” vs. “Company” should matter much less to BERT than to TF-IDF.
  • Semantic grouping: sentences 3 and 4 (the merchant pair) should be very similar, and both should be closer to the Virginia Company sentences (all about trade) than to sentence 5 (harvest) or 6 (Parliament).

Part VI — Side-by-Side: The Virginia Company Trio

Let’s isolate the three Virginia Company sentences to really see the difference between TF-IDF and BERT.

```{python}
trio = [
    "The Virginia Companie did advance traffique beyond the seas.",
    "The Virginia Company did advance traffic beyond the seas.",
    "The Virginia Company promoted overseas trade."
]
trio_labels = ["Archaic spelling", "Modern spelling", "Paraphrase"]

# TF-IDF
tfidf_trio = vectorizer.fit_transform(trio)
tfidf_trio_sim = cosine_similarity(tfidf_trio)

# BERT
bert_trio = model.encode(trio)
bert_trio_sim = cosine_similarity(bert_trio)

print("=== Virginia Company Trio ===\n")
print("TF-IDF Similarity:")
for i in range(3):
    for j in range(3):
        print(f"  {trio_labels[i]:20s} ↔ {trio_labels[j]:20s} : {tfidf_trio_sim[i][j]:.3f}")
    print()

print("BERT Similarity:")
for i in range(3):
    for j in range(3):
        print(f"  {trio_labels[i]:20s} ↔ {trio_labels[j]:20s} : {bert_trio_sim[i][j]:.3f}")
    print()
```

This comparison drives the point home: TF-IDF measures word overlap, BERT measures meaning overlap. For historical text — where spelling varies, vocabulary changes, and meaning persists — this distinction is crucial.

```{python}
# Fit PCA to reduce from 384 dimensions to 2
pca = PCA(n_components=2)
```

Part VIII — Semantic Search (Retrieval)

  • Create file: step4_semantic_search.py — the retrieval/search demo (Part VIII)

One practical application of sentence embeddings is semantic search: given a query, find the most similar sentence in a corpus.

```{python}
query = "The Virginia Company expanded trade overseas."

query_embedding = model.encode([query])
similarities = cosine_similarity(query_embedding, embeddings)[0]

print(f"Query: \"{query}\"\n")
print("Ranked results:")
print("-" * 60)

# Sort by similarity (descending)
ranked = sorted(enumerate(similarities), key=lambda x: x[1], reverse=True)

for rank, (idx, sim) in enumerate(ranked, 1):
    print(f"  {rank}. [{sim:.3f}] {sentences[idx]}")
```

This is the same principle behind modern search engines and retrieval-augmented generation (RAG) systems. BERT encodes the meaning of the query and finds the closest matches by meaning, not by keyword overlap.

Part IX — [CLS] Pooling vs. Mean Pooling–DIDN’T USE

Earlier in the tutorial we used the sentence-transformers library to produce sentence embeddings. That library hides an important point: BERT’s outputs token-level vectors, but we needed to convert them into a single vector representing the entire sentence.

When BERT processes a sentence, it does not produce a single embedding. Instead, it produces a vector for every token, where each vector captures the meaning of that token in context. That’s great, but for an application like semantic search (part VIII above), we needed a single vector for the whole sentence. This requires a pooling strategy.

The sentence-transformers library handled embedding extraction for us. But under the hood, there are choices being made about how to convert BERT’s token-level outputs into a single sentence vector. Let’s look at two common strategies.

When BERT processes a sentence, it produces a 768-dimensional vector for every token. To get a single sentence-level embedding, we need to aggregate:

  • [CLS] pooling: Use the vector for the special [CLS] token that BERT prepends to every sentence. During pre-training, this token was trained to capture whole-sentence information.

  • Mean pooling: Average the vectors for all tokens in the sentence. This often works better in practice because it incorporates information from every position.

  • Create file: step5_pooling.py — [CLS] vs. mean pooling (Part IX)

The sentence-transformers library handled embedding extraction for us. But under the hood, there are choices being made about how to convert BERT’s token-level outputs into a single sentence vector. Let’s look at two common strategies.

When BERT processes a sentence, it produces a 768-dimensional vector for every token. To get a single sentence-level embedding, we need to aggregate:

  • [CLS] pooling: Use the vector for the special [CLS] token that BERT prepends to every sentence. During pre-training, this token was trained to capture whole-sentence information.

  • Mean pooling: Average the vectors for all tokens in the sentence. This often works better in practice because it incorporates information from every position.

```{python}
# Load the base BERT model directly (not sentence-transformers)
tokenizer = BertTokenizer.from_pretrained("bert-base-uncased")
bert_model = BertModel.from_pretrained("bert-base-uncased")
bert_model.eval()

def get_cls_embedding(sentence):
    """Extract the [CLS] token embedding."""
    inputs = tokenizer(sentence, return_tensors="pt", truncation=True, max_length=512)
    with torch.no_grad():
        outputs = bert_model(**inputs)
    # [CLS] is always the first token
    return outputs.last_hidden_state[0, 0, :].numpy()

def get_mean_embedding(sentence):
    """Average all token embeddings (excluding special tokens)."""
    inputs = tokenizer(sentence, return_tensors="pt", truncation=True, max_length=512)
    with torch.no_grad():
        outputs = bert_model(**inputs)
    # Get attention mask to exclude padding
    mask = inputs["attention_mask"][0].unsqueeze(-1)
    hidden = outputs.last_hidden_state[0]
    # Mean over non-padding tokens
    return (hidden * mask).sum(dim=0).numpy() / mask.sum().item()

# Compare the two methods
test_pair = [
    "The Virginia Company promoted overseas trade.",
    "The harvest was plentiful this year."
]

cls_embs = [get_cls_embedding(s) for s in test_pair]
mean_embs = [get_mean_embedding(s) for s in test_pair]

cls_sim = cosine_similarity([cls_embs[0]], [cls_embs[1]])[0][0]
mean_sim = cosine_similarity([mean_embs[0]], [mean_embs[1]])[0][0]

print(f"Sentence A: \"{test_pair[0]}\"")
print(f"Sentence B: \"{test_pair[1]}\"")
print()
print(f"[CLS] pooling similarity:  {cls_sim:.4f}")
print(f"Mean pooling similarity:   {mean_sim:.4f}")
```

The exact numbers will vary, but you should see that both methods produce a low similarity for these unrelated sentences. The key insight is that the choice of pooling strategy matters — and that sentence-transformers models are specifically fine-tuned for mean pooling to work well.

Part X — What BERT “Knows”: Masked Language Modeling

  • Create file: step6_mlm.py — masked language modeling, including the Geneva Bible preview (Parts X–XI)

BERT was trained to predict masked words. We can use this to probe what the model has learned. This is called masked language modeling (MLM) — and it will become very important next week when we fine-tune BERT on historical text.

```{python}
from transformers import pipeline

fill_mask = pipeline("fill-mask", model="bert-base-uncased")

print("=== Masked Language Modeling ===\n")
print("BERT predicts the most likely word for [MASK]:\n")

test_sentences = [
    "The Virginia Company traded [MASK] across the Atlantic.",
    "The merchants of London bought and sold [MASK].",
    "The king granted a royal [MASK] to the trading company.",
]

for sent in test_sentences:
    print(f"Input: {sent}")
    results = fill_mask(sent)
    for r in results[:3]:  # Top 3 predictions
        print(f"  → {r['token_str']:15s}  (score: {r['score']:.3f})")
    print()
```

Why this matters

These predictions reveal what BERT “knows” from its training data. Notice that BERT predicts modern words in modern-feeling contexts. But what happens when we give it Early Modern English?

```{python}
print("=== MLM on Early Modern English ===\n")

historical_tests = [
    "The companie did [MASK] traffique beyond the seas.",
    "The merchants had [MASK] in silkes and spices.",
    "By the grace of God the [MASK] was plentiful.",
]

for sent in historical_tests:
    print(f"Input: {sent}")
    results = fill_mask(sent)
    for r in results[:3]:
        print(f"  → {r['token_str']:15s}  (score: {r['score']:.3f})")
    print()
```

BERT often struggles here. It was trained on modern text (Wikipedia, contemporary books), so Early Modern spellings and syntax are outside its comfort zone. The predictions may be plausible but slightly off, or the confidence scores may be noticeably lower.

This is the motivation for next week: if we fine-tune BERT on historical text, we can teach it to be better at understanding — and predicting — the language of our period.

Part XI — A Preview: BERT Meets the Geneva Bible

To close out this week, let’s give BERT a taste of what is coming. We will feed it a few sentences from the Geneva Bible (1560) and see how it handles the archaic English.

```{python}
geneva_samples = [
    "In the beginning God created the heauen and the earth.",
    "And the earth was without forme and voide, and darkenesse was vpon the deepe.",
    "Then God saide, Let there be light: And there was light.",
    "And God sawe the light that it was good, and God separated the light from the darkenes.",
]

print("=== Geneva Bible — MLM Predictions ===\n")

# Mask one word in each sentence
masked_geneva = [
    "In the [MASK] God created the heauen and the earth.",
    "And the earth was without forme and [MASK], and darkenesse was vpon the deepe.",
    "Then God saide, Let there be [MASK]: And there was light.",
    "And God sawe the light that it was [MASK], and God separated the light from the darkenes.",
]

for original, masked in zip(geneva_samples, masked_geneva):
    print(f"Original:  {original}")
    print(f"Masked:    {masked}")
    results = fill_mask(masked)
    for r in results[:3]:
        print(f"  → {r['token_str']:15s}  (score: {r['score']:.3f})")
    print()
```

Some predictions will be reasonable (BERT knows biblical language from its training data), but others will miss the mark — especially for archaic vocabulary. Next week, we will fine-tune BERT on the full Geneva Bible to fix this.
